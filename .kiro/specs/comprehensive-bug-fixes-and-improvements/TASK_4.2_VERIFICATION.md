### 2. Download Functionality ✅

**Feature**: Download button to save QR code as PNG image

**Implementation**:
```typescript
const handleDownloadQR = () => {
  if (!paymentDetails.qrCodeBase64) return;

  const link = document.createElement("a");
  const qrDataUrl = paymentDetails.qrCodeBase64.startsWith("data:")
    ? paymentDetails.qrCodeBase64
    : `data:image/png;base64,${paymentDetails.qrCodeBase64}`;
  
  link.href = qrDataUrl;
  link.download = `qris-payment-${Date.now()}.png`;
  document.body.appendChild(link);
  link.click();
  document.body.removeChild(link);
};
```

**Button Design**:
- Styled with gold theme (matching project design system)
- Download icon (SVG) + "Download QR Code" text
- Hover effect: background opacity increases
- Only shown when `qrCodeBase64` exists
- File naming: `qris-payment-<timestamp>.png`

**Browser Compatibility**:
- Uses native `<a>` element with `download` attribute
- Works with both `data:image/png;base64,<data>` and raw base64 strings
- Supported in all modern browsers (Chrome, Firefox, Safari, Edge)

## Known Issues & Considerations

### 1. Base64 Image Size
**Consideration**: Base64 QR codes can be large (10-50KB)

**Mitigation**:
- QR codes are typically small (220x220px)
- Backend should optimize QR generation (low DPI for QRIS)
- Consider switching to PNG URL endpoint if size becomes issue

**Recommendation**: Current approach is fine for MVP

### 2. Download Filename
**Current**: Uses timestamp: `qris-payment-1718380800000.png`

**Consideration**: Could be more user-friendly

**Alternatives**:
- `qris-<transactionId>.png`
- `qris-<amount>cc.png`
- `emerald-kingdom-qris-<date>.png`

**Recommendation**: Current approach is acceptable, can be enhanced later

## Integration with Parent Component

The QRISPaymentDisplay component is used in `topup/page.tsx`:

```typescript
{selectedMethod === "QRIS" && (
  <QRISPaymentDisplay
    paymentDetails={paymentData.paymentDetails || {}}
    amount={selectedPackage.cc}
    fiatAmount={selectedPackage.price}
    expiresAt={paymentData.expiresAt}
  />
)}
```

**Data Flow**:
1. User selects QRIS payment method
2. Parent calls `initiatePaymentAction` → Backend POST `/payment/initiate`
3. Backend returns `paymentData` with `paymentDetails.qrCodeBase64`
4. Parent renders QRISPaymentDisplay with payment data
5. Component displays QR code with zoom/download
6. Parent component polls for payment status updates via `usePaymentSocket`
